## 🔄 HOW IT WORKS

### **Scheduled Posts Flow:**
```
User schedules post with prompt:
"Create a post on cloud deployment with diagram workflow"
        ↓
Dashboard saves to database (raw content)
        ↓
Scheduler runs every 60 seconds
        ↓
At scheduled time:
  1. Claude generates professional content
  2. Claude generates Mermaid diagram code
  3. Python converts Mermaid → PNG
  4. Creates approval file with text + diagram
        ↓
Human approves (moves to Approved/)
        ↓
Posts with image attached ✅
```

### **Normal Posts Flow (via Orchestrator):**
```
User creates post via dashboard
        ↓
Task created in Needs Action/
        ↓
Orchestrator detects Twitter/Facebook task
        ↓
Orchestrator uses Claude to generate content
        ↓
Creates approval file
        ↓
Human approves
        ↓
Posts to social media ✅
```

## ⚠️ CURRENT ISSUE

### **Problem:**
When you schedule a post with prompt like:
```
create a post on cloud deployment with diagram workflow and explaining it as an expert teacher of devops #devops
```

The approval file shows **raw content** without Claude enhancement or diagram.

### **Root Cause:**
The **scheduler hasn't run yet** or **scheduled time hasn't arrived**.

The scheduler is designed to enhance content **AT THE SCHEDULED TIME**, not when you first schedule it.

### **Solution:**
1. **Start the scheduler:**
   ```bash
   python scheduler/main_scheduler.py
   ```

2. **Schedule a post for 1-2 minutes from now**

3. **Wait for scheduled time** - scheduler will:
   - Detect it's time to process
   - Call Claude to enhance content
   - Generate diagram
   - Create approval file with enhanced content + image
